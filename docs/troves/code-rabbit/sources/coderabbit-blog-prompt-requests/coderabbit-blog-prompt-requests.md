# Show me the prompt: What to know about prompt requests

## Key Points
- **Prompt Requests**: In AI-assisted development, "show me the prompt" is becoming as important as "show me the code." Prompt requests ensure alignment and best practices early, before any code is generated or committed.
- **Intent Capture**: Prompt requests help teams align on what should be built, define boundaries, and constrain agent behavior. This is critical because large language models (LLMs) are non-deterministic, and variability is highest upstream.
- **Pull Requests**: Pull requests exist because software development ultimately involves shipping specific, deterministic artifacts into production. Teams need a concrete mechanism to review, test, audit, and approve the exact code that runs.
- **Future of Review**: The more realistic future is not "prompt requests instead of pull requests," but a hybrid approach where both are used to ensure quality and accountability.

## Accountability Shift
- As more code is generated by LLMs, accountability does not disappear—it moves upstream.
- The question facing modern engineering teams is not whether AI-generated code can be reviewed, but **where and how review should happen** when intent is increasingly expressed before code exists at all.

## Industry Context
- **Quote**: Gergely Orosz (Pragmatic Engineer) shared a quote from Peter Steinberger (creator of Clawdbot) suggesting that traditional pull requests may no longer be the best way to review changes as more code is produced with LLMs. Instead, reviewers should be given the prompt that generated the change.
